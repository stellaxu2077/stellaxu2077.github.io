# 工具篇
1. Github
- 开源社区，每人都能创建自己的开源代码项目，并对别人的项目发表issue，包括bug和feature idea
- 可以复制项目（Fork）然后修改，给原项目发送PullRequest，原作者若认为贡献有用，会将代码merge

2. Stackoverflow
- 问答社区

3. 搜索引擎
- 广告少、搜索结果公正、英文搜索质量好
- Google, Bing, Quark（移动端）国际搜索不够好

4. 浏览器
- Chrome, Edge, 夸克
- 插件使得浏览器强大，在Chrome前端开发者工具帮助下，直接在浏览器中进行前端开发!

# 观念篇

> **身份认同：把自己当成开发者**  

> **找到教程后学习，理解知识的整体框架（systematically），实战填补细节，写技术文章发表（费曼学习法）
> 先学最基本的知识，马上开始开发实践，以不断遇到的问题为中心延伸扩展学习，善用搜索和提问平台（包括ReferenceDoc即文档，成品说明书，效果比教程好，因为不过时），完成细节知识的补充   
不要害怕学习注意力的分散！**

> “知识用进废退，忘就忘了。”——把注意力放在实战成果即开发上

要在一段时间内通过学习某语言，运用它做出一个小项目
学习语言前先了解它适合用来做什么→定目标（时间+成果）
e.g. 
	大多数游戏用C++
	python适合数据分析、爬虫、AI
	Java适合web后端开发
	前端最好入门，html，css，JavaScript学起来容易

学习的最终导向是成就感与反馈。
尤其是实践性强的东西，很容易使之变有趣。

简历：Github账号XX   XX项目作者

新手如何开始实践？
克隆别人的项目→修改别人的项目→构建自己的项目
尽力使学习曲线平缓，先找已有的类似想法进行克隆（e.g. 在GitHub搜索小游戏），阅读代码过程中有不懂就去查，最终看懂别人的项目，然后就可以尝试修改（二次开发），比凭空写难度低，最后完全写成自己的东西


希望用C写一个有图形界面的小游戏！

要选一流的东西
找被很多人认可的项目
一定要学会写技术文章发表！
看不懂，不要怀疑自己的学习能力，可能是文章讲得不好（换一篇），或者缺少前置知识（去学习）
应当自行搜索最好的教材
